데이터 모델은 어떻게 정의하나요?

Express는 데이터베이스와 관련된 기능들을 전혀 갖고 있지 않으며, 이는 third-party 모듈에 의존합니다. 따라서 거의 모든 데이터베이스의 인터페이스 사용을 가능하게 하지만, 데이터 모델은 각 모듈에 따라 작성해야합니다.

어떻게 인증을 처리하나요?

이 또한 Express가 스스로 지원하는 부분은 아니므로 사용하고 싶은 인증 솔루션을 사용하시면 됩니다. 간단한 username / password 스킴을 보기 위해서는 example를 참고하시기 바랍니다.

Express가 지원하는 템플릿 엔진은 무엇인가요?

(path, locals, callback) signature 형태를 가진 것이라면 어떤 것이든 지원 가능합니다. 템플릿 엔진과 캐싱을 normalize 하기 위해서, consolidate.js 프로젝트를 확인해 볼 것을 권합니다. 리스트에 포함 되지 않은 템플릿 엔진도 Express signature를 지원할 가능성이 높습니다.

제 애플리케이션을 어떻게 구조화 시켜야 하나요?

이 질문에 대한 명확한 답은 없습니다. 이는 애플리케이션의 규모와, 관련된 팀에 달려있기 때문입니다. 최대한의 유연함을 지키기 위해 Express는 애플리케이션 구조에 관해서는 따로 예외사항을 두지 않습니다.

라우트들이나 구체적인 애플리케이션 로직은 원하는 디렉토리 구조안에 원하는 만큼 많이 포함시킬 수 있습니다. 아래 예제들을 참고해보세요.

  • Route listings
  • Route map
  • Route bootstrapping
  • MVC style controllers

이러한 패턴들을 단순화 시키는 Express의 third-party 익스텐션 또한 사용 가능합니다.

  • Resourceful routing
  • Namespaced routing

여러 디렉토리에 저장된 static 파일들을 어떻게 제공하나요?

일반적으로 애플리케이션 안에서 미들웨어를 여러번 사용하면 됩니다. 

app.use(express.static('public'));
app.use(express.static('files'));

404를 어떻게 처리 해야 하나요?

Express에서 404는 에러의 결과로써 존재하지 않습니다. 그러므로 오류 핸들러 미들웨어는 404를 처리하지 않으며, 404를 처리 하고자 한다면, 미들웨어들의 가장 아래에 404를 처리하는 또다른 미들웨어를 하나 더 추가하시면 됩니다. 

app.use(function(req, res, next){
  res.send(404, 'Sorry cant find that!');
});

Express에서 오류 핸들러는 어떻게 설정해야 하나요?

오류 처리 미들웨어는 다른 보통 미들웨어와 마찬가지로 정의되지만, 특히 오류 처리 미들웨어는 4개의 argument (err, req, res, next)로 정의되어야 합니다. 

app.use(function(err, req, res, next){
  console.error(err.stack);
  res.send(500, 'Something broke!');
});

더 많은 정보는 error-handling를 참고하세요.
